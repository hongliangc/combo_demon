# 文档组织与重构 Skill

> **触发词**: 文档重组, 文档整理, 架构文档, 模块化文档
> **用途**: 按功能拆分和整理项目文档，优化 Token 消耗

---

## 核心原则

1. **按功能分类**：每个文档只讲一个主题（状态机、战斗、组件等）
2. **Token 控制**：总览 ~600 tokens，专项文档 ~800-1500 tokens
3. **单一来源**：详细内容只在一处说明，其他地方用链接引用
4. **及时归档**：已完成的任务文档移至 `archive/`，不留在活跃目录

## 重组流程

1. **分析**：查找重复内容和过长文档（>2000 tokens）
2. **拆分**：按功能创建独立文档，数字前缀排序（00_, 01_, 02_...）
3. **索引**：创建/更新总览索引文档
4. **清理**：过时文档归档，更新引用链接，删除重复内容

## 文档结构规范

```
architecture/
├── 00_overview.md          # 总览索引（必需）
├── 01_[module].md          # 各模块独立文档
└── ...
```

## 检查清单

- [ ] 每个文档 ≤ 1500 tokens
- [ ] 无重复内容跨文件
- [ ] 所有文档在索引中有入口
- [ ] 过时文档已归档
