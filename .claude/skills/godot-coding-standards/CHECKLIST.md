# Godot 代码审查检查清单

## 通用性检查
- [ ] 是否使用 `@export` 使组件可配置？
- [ ] 是否避免硬编码路径和值？
- [ ] 是否可以在不同场景中复用？
- [ ] 是否使用信号进行松耦合？

## 模块化检查
- [ ] 是否遵循单一职责原则？
- [ ] 是否使用组件而非巨型类？
- [ ] 是否通过信号而非直接调用通信？
- [ ] 避免神类 (God Class)？

## 可复用性检查
- [ ] 是否有清晰的公共接口？
- [ ] 是否有文档注释说明用途？
- [ ] 是否可以独立使用而不依赖特定场景？
- [ ] Resource 类是否正确使用？

## 代码质量检查
- [ ] 是否所有变量和函数都有类型注解？
- [ ] 是否使用了正确的命名规范？
  - 类名：PascalCase
  - 变量/函数：snake_case
  - 常量：UPPER_SNAKE_CASE
- [ ] 是否有必要的错误检查？
- [ ] 是否避免了性能陷阱？
  - 缓存节点引用 (@onready)
  - 避免循环中查找节点
  - 预计算常量

## 文档检查
- [ ] 是否有类级文档注释（##）？
- [ ] 复杂方法是否有参数和返回值说明？
- [ ] 是否使用分隔注释组织代码结构？

## 信号规范检查
- [ ] 信号声明在文件顶部？
- [ ] 使用 Godot 4 风格连接？ (`signal.connect(method)`)
- [ ] 信号处理函数使用 `on_` 前缀？

## Export 变量检查
- [ ] 使用 `@export_group` 分组？
- [ ] 有明确的类型注解？
- [ ] 提供合理的默认值？

## 代码组织顺序
1. extends / class_name
2. 类文档注释
3. 信号
4. 枚举和常量
5. @export 变量
6. public 变量
7. private 变量 (_前缀)
8. @onready 变量
9. 内置回调 (_ready, _process 等)
10. 公共方法
11. 私有方法 (_前缀)
12. 信号处理函数 (on_前缀)
